## 触发提示词（你可以直接复制）

- “帮我写一个作品集项目介绍（招聘向），技术栈是 React + Tailwind + Node，项目叫 XXX。”
- “把这段项目描述改得更像面试官爱看的，保留关键信息，别堆 buzzword。”
- “给我的 About me 写精简版和完整版，偏前端工程化方向。”
- “为作品集首页写 3 个标题 + 3 个副标题，风格分别是：克制专业 / 稍微大胆 / 极简。”
- “给这个项目写 case study，按 Problem/Approach/Implementation/Results/Learnings。”

## 项目一行简介（3 个风格）

输入要点：
- 项目做什么：
- 用户是谁：
- 差异化：
- 结果/影响：

输出示例：
- **克制**：为【目标用户】构建【产品/系统】，用【差异化】将【指标】提升【数值】。
- **大胆**：把【痛点】用【关键手段】解决了：让【指标】从【A】到【B】。
- **极简**：【做什么】｜【差异化】｜【结果】

## 招聘向项目描述（可直接替换括号）

**项目背景**：为（谁）解决（什么问题），目标是（业务/体验指标）。

**我的职责**
- 负责（模块/页面/流程）设计与实现
- 搭建（工程化/组件库/CI）以提升（效率/一致性）
- 优化（性能/可访问性/稳定性），定位并修复（具体问题）

**关键挑战**
- （约束 1：时间/资源/兼容性/遗留系统）
- （约束 2：数据/质量/协作）

**解决方案**
- 采用（方案 A）而非（方案 B），因为（原因/取舍）
- 将（复杂问题）拆分为（子问题），优先处理（关键路径）
- 用（监控/埋点/自动化测试）保障上线质量

**结果与影响**
- （指标提升/故障率下降/首屏时间/转化率）+（数值/区间）
- （覆盖范围/上线版本/用户量/可验证事实）

**技术栈**：React / TypeScript / Tailwind / （其他）

## 作品集首页信息架构（建议）

- Hero：一句话定位 + 1 行证据 + CTA（查看项目/联系）
- Projects：3–6 个精选项目（每个：一行简介 + 3 个亮点）
- About：80–140 字精简版
- Skills：按能力模块（工程化/性能/交互/后端）
- Contact：邮箱/LinkedIn/GitHub
