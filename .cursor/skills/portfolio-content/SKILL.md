# Portfolio Content (中文)

## Quick start

When asked to write or improve portfolio content, do this:

1. Ask for (or infer) the target audience and goal (recruiter, client, interview, conversion).
2. Pick an output type (see templates below) and keep it skimmable.
3. Prefer outcomes and evidence over tasks:
   - Impact metrics (%, time saved, revenue, users), constraints, trade-offs, why decisions were made.
4. Add light SEO:
   - One primary keyword + 3–6 supporting keywords, used naturally in headings and first paragraph.
5. Provide 2–3 variations when tone may differ (professional / bold / concise).

## Output templates

### Project one-liner

- Structure: **[What] + [For whom] + [Differentiator] + [Outcome]**
- 1 sentence, 20–40 Chinese characters preferred, no buzzword stacking.

### Project description (招聘向)

Use this layout:

- **项目背景**：一句话说明为何做
- **我的职责**：3–5 条，动词开头
- **关键挑战**：2–3 条（性能/协作/约束/不确定性）
- **解决方案**：3–6 条，突出决策与取舍
- **结果与影响**：量化优先；无数据则用可验证事实（如上线范围、稳定性、可维护性）
- **技术栈**：只列与卖点相关的

### Case study (更完整)

Use this layout:

- **Problem**：用户痛点/业务目标/约束
- **Approach**：拆解、方案比较、关键决策
- **Implementation**：核心模块、架构图（可用文字版）、质量保障（测试/监控）
- **Results**：指标 + 影响面
- **Learnings**：2–4 条，面向下一次迭代

### “About me” (中文)

Give 2 versions:

- **精简版**：80–140 字
- **完整版**：200–320 字

Include:
- 定位（前端/全栈/方向）
- 1–2 个差异化能力（性能/工程化/设计系统/产品意识/沟通协作）
- 可信度（年限/领域/代表项目/可量化成果）
- 轻 CTA（欢迎交流/查看项目/联系）

## SEO checklist (轻量)

- 先确定 **主关键词**（1 个）与 **支持关键词**（3–6 个）。
- 主关键词出现在：标题/首段/至少一个二级标题。
- 支持关键词自然分布，不堆叠。
- 若提供中英双语页面：关键词以中文为主，英文补充同义词。

## Style guide (中文写作)

- Use short paragraphs; max 2–3 lines per paragraph.
- Prefer numbers and nouns over adjectives.
- Avoid empty claims (e.g., “高性能/高可用/可扩展”) unless backed by evidence.
- Use consistent tense and terminology across the site.

## When information is missing

If details are missing, proceed by:

1. Drafting a best-effort version using placeholders like `【指标】` and `【约束】`.
2. Listing **exactly 5** follow-up questions that unblock the next iteration (no more).
